# HireAI MVP Features

## Общая информация

Документ описывает функциональные возможности (Features) MVP платформы HireAI.

Документ не содержит технической реализации и используется как источник требований для разработки.

---

# F-001 Landing Page

## Описание

Одностраничный сайт является точкой входа в систему.

### Возможности

* информация о продукте;
* кнопка регистрации;
* кнопка входа;
* переход к авторизации.

---

# F-002 Authentication

## Описание

Система поддерживает регистрацию и авторизацию пользователей.

### Возможности

* регистрация нового пользователя;
* вход в систему;
* выход из системы;
* восстановление сессии;
* проверка авторизации.

---

# F-003 Role Based Access

## Описание

После входа пользователь получает доступ только к разрешённым функциям.

### Поддерживаемые роли

* Administrator
* HR Operator
* Manager
* Candidate

---

# F-004 Enterprise Isolation

## Описание

Все данные принадлежат конкретному предприятию.

Каждый пользователь работает только внутри своего предприятия.

### Ограничения

Пользователь не может:

* видеть данные других предприятий;
* изменять данные других предприятий;
* получать уведомления другого предприятия.

---

# F-005 Candidate Registration

## Описание

Кандидат может зарегистрироваться самостоятельно.

После регистрации создаётся учётная запись.

---

# F-006 Candidate Questionnaire

## Описание

После регистрации кандидат заполняет кадровую анкету и отправляет ее как JSON.

Анкета соответствует структуре личного листка по учету кадров (ЛПК).

### Возможности

* сохранение анкеты;
* редактирование анкеты до отправки;
* просмотр введённых данных.
* запуск AI ranking после сохранения;

### DoD

* анкета принимается как JSON payload;
* после сохранения создается или обновляется candidate profile;
* статус кандидата устанавливается в `Unassigned`.

---

# F-007 HR Candidate Registration

## Описание

HR Operator может загрузить JSON-анкету и создать карточку кандидата вручную.

Используется в случаях, когда кандидат присутствует лично и кадровый специалист переносит данные из бумажной анкеты.

### DoD

* система принимает JSON анкеты;
* запись кандидата попадает в пул предприятия;
* запускается AI ranking;
* создается уведомление.

---

# F-008 Candidate Pool

## Описание

После сохранения анкеты кандидат появляется в общем списке предприятия.

### Начальный статус

Unassigned

### Возможности

* просмотр списка;
* поиск;
* фильтрация;
* открытие карточки кандидата.
* просмотр ranking score и best vacancy match;

---

# F-009 Vacancy Management

## Описание

Administrator управляет вакансиями предприятия.

### Возможности

* создание вакансии;
* редактирование вакансии;
* удаление вакансии;
* просмотр вакансий;
* изменение статуса вакансии.

### DoD

* CRUD доступен только administrator;
* изменения ограничены текущим enterprise_id;
* после создания вакансии может быть создано уведомление.

---

# F-010 Vacancy Directory

## Описание

Все пользователи предприятия могут просматривать открытые вакансии в соответствии со своими правами доступа.

Manager и HR Operator видят только данные своего предприятия.

---

# F-011 Candidate Assignment

## Описание

Administrator закрепляет кандидата за вакансией.

После назначения обновляется статус кандидата.

### Результат

* кандидат связан с вакансией;
* вакансия отображает назначенного кандидата;
* менеджер получает актуальную информацию.
* кандидат получает статус `Assigned`.

### DoD

* закрепление доступно только administrator;
* нельзя закрепить кандидата другого enterprise;
* создается уведомление менеджерам.

---

# F-012 Candidate Status

## Описание

Для каждого кандидата отображается текущий этап обработки.

### MVP статусы

* Unassigned
* Assigned

После MVP список может быть расширен.

# F-019 Tenant Isolation

## Описание

Каждый пользователь и каждая сущность привязаны к одному `enterprise_id`.

### Ограничения

* пользователь не видит данные другого предприятия;
* кандидат не получает доступ к чужим вакансиям;
* manager и HR Operator работают только в своем enterprise;
* AI ranking рассчитывается только внутри текущего enterprise.

---

# F-020 AI Ranking

## Описание

Система ранжирует кандидатов локально с помощью Qwen 2.5.

### Возможности

* анализ JSON-анкеты кандидата;
* расчет score от 0 до 10;
* определение лучшей вакансии;
* формирование ranking list по всем вакансиям предприятия.

### DoD

* ranking запускается после сохранения анкеты;
* score лежит в диапазоне 0-10;
* данные не покидают инфраструктуру предприятия.

# Feature Roadmap (Post-MVP)

Следующие возможности не входят в MVP:

* AI-анализ кандидатов;
* автоматический подбор кандидатов;
* ранжирование кандидатов;
* рекомендации по найму;
* история изменений;
* аналитическая панель;
* экспорт данных;
* интеграция с кадровыми системами предприятия;
* email-уведомления;
* мобильная версия;
* REST API для внешних систем.
